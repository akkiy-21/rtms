# Requirements Document

## Introduction

このドキュメントは、アラームグループのアラームアドレス設定において、CSVファイルを取り込んだ後にユーザーがパース範囲や設定を調整できる機能の要件を定義します。現在のシステムではフロントエンドでCSVを自動パースしていますが、様々なCSV形式に対応するため、ユーザーが手動で設定を調整できるインターフェースが必要です。

## Glossary

- **CSV Parser**: CSVファイルを解析してデータを抽出するシステム
- **Parse Range**: CSVファイル内でデータとして扱う行と列の範囲
- **Alarm Address**: PLCのアラーム情報を格納するメモリアドレス
- **Alarm Group**: 関連するアラームアドレスをまとめたグループ
- **Column Mapping**: CSVの列とデータベースフィールドの対応関係
- **Preview Table**: パース結果をユーザーに表示するためのテーブル
- **Header Row**: CSVファイルの列名を含む行
- **Data Rows**: 実際のデータを含む行
- **Extraction Rule**: CSVデータからアラーム情報を切り出すためのルール定義
- **Regex Pattern**: データ抽出に使用する正規表現パターン
- **Output Mapping**: 抽出されたデータをアラームアドレスフィールドにマッピングする設定

## Requirements

### Requirement 1

**User Story:** アラーム管理者として、CSVファイルをアップロードした後にパース結果をプレビューしたい。これにより、データが正しく解析されているかを確認できる。

#### Acceptance Criteria

1. WHEN ユーザーがCSVファイルをアップロードする THEN システムは初期パース結果を表形式で表示する
2. WHEN パース結果が表示される THEN システムは行番号と列番号を含むグリッド形式で表示する
3. WHEN CSVに日本語と英語の混在データがある THEN システムは文字エンコーディングを自動検出して正しく表示する
4. WHEN 大きなCSVファイルがアップロードされる THEN システムは最初の100行のみをプレビューとして表示する
5. WHEN プレビューテーブルが表示される THEN システムはスクロール可能な形式で全ての列を表示する

### Requirement 2

**User Story:** アラーム管理者として、CSVのヘッダー行を指定したい。これにより、データ行とヘッダー行を正しく区別できる。

#### Acceptance Criteria

1. WHEN プレビューテーブルが表示される THEN システムはヘッダー行選択のドロップダウンを提供する
2. WHEN ユーザーがヘッダー行を変更する THEN システムは即座にプレビューテーブルを更新する
3. WHEN ヘッダー行が設定される THEN システムは選択された行をヘッダースタイルで表示する
4. WHEN ヘッダー行が「なし」に設定される THEN システムは列番号（A, B, C...）をヘッダーとして使用する
5. WHEN 複数のヘッダー行候補がある THEN システムは最初の非空白行をデフォルトとして選択する

### Requirement 3

**User Story:** アラーム管理者として、データとして使用する行範囲を指定したい。これにより、不要な行を除外してデータをインポートできる。

#### Acceptance Criteria

1. WHEN プレビューテーブルが表示される THEN システムは開始行と終了行を指定する入力フィールドを提供する
2. WHEN ユーザーが行範囲を変更する THEN システムは指定範囲外の行をグレーアウト表示する
3. WHEN 行範囲が設定される THEN システムは範囲内の行数をカウント表示する
4. WHEN 無効な行範囲が入力される THEN システムはエラーメッセージを表示し、前の有効な値に戻す
5. WHEN 行範囲がヘッダー行と重複する THEN システムは警告メッセージを表示する

### Requirement 4

**User Story:** アラーム管理者として、CSVの列とデータベースフィールドのマッピングを設定したい。これにより、異なるCSV形式に対応できる。

#### Acceptance Criteria

1. WHEN プレビューテーブルが表示される THEN システムは各列に対してフィールドマッピングのドロップダウンを提供する
2. WHEN 列マッピングが変更される THEN システムは変更を即座にプレビューに反映する
3. WHEN 必須フィールドがマッピングされていない THEN システムは警告アイコンを表示する
4. WHEN 同じフィールドが複数の列にマッピングされる THEN システムはエラーメッセージを表示する
5. WHEN マッピング設定が完了する THEN システムは設定内容をサマリー表示する

### Requirement 5

**User Story:** アラーム管理者として、パース設定を保存して再利用したい。これにより、同じ形式のCSVファイルを効率的に処理できる。

#### Acceptance Criteria

1. WHEN パース設定が完了する THEN システムは設定をテンプレートとして保存するオプションを提供する
2. WHEN 設定テンプレートが保存される THEN システムはテンプレート名の入力を求める
3. WHEN 新しいCSVをアップロードする THEN システムは保存済みテンプレートの選択肢を提供する
4. WHEN テンプレートが選択される THEN システムは保存された設定を自動適用する
5. WHEN テンプレートが適用される THEN システムはユーザーが設定を微調整できるようにする

### Requirement 6

**User Story:** アラーム管理者として、パース結果をバリデーションしたい。これにより、データの整合性を確保してからインポートできる。

#### Acceptance Criteria

1. WHEN パース設定が完了する THEN システムはデータバリデーションを実行する
2. WHEN 必須フィールドに空の値がある THEN システムはエラー行をハイライト表示する
3. WHEN データ型が不正な値がある THEN システムは該当セルを赤色でマーク表示する
4. WHEN 重複するアラーム番号がある THEN システムは重複行を警告表示する
5. WHEN バリデーションエラーがある THEN システムはインポートボタンを無効化する

### Requirement 7

**User Story:** アラーム管理者として、設定したパース結果でデータをインポートしたい。これにより、正しく設定されたデータをシステムに取り込める。

#### Acceptance Criteria

1. WHEN バリデーションが成功する THEN システムはインポートボタンを有効化する
2. WHEN インポートボタンがクリックされる THEN システムは確認ダイアログを表示する
3. WHEN インポートが実行される THEN システムは進行状況を表示する
4. WHEN インポートが完了する THEN システムは成功メッセージと処理件数を表示する
5. WHEN インポートでエラーが発生する THEN システムはエラー詳細とロールバック情報を表示する

### Requirement 8

**User Story:** アラーム管理者として、異なるCSV形式に対応したい。これにより、様々なPLCメーカーやツールから出力されたCSVファイルを処理できる。

#### Acceptance Criteria

1. WHEN 区切り文字が異なるCSVがアップロードされる THEN システムは区切り文字（カンマ、セミコロン、タブ）を自動検出する
2. WHEN 自動検出が失敗する THEN システムは手動で区切り文字を選択するオプションを提供する
3. WHEN 引用符で囲まれたフィールドがある THEN システムは引用符を適切に処理する
4. WHEN 改行を含むフィールドがある THEN システムは複数行フィールドを正しく解析する
5. WHEN 異なる文字エンコーディングのファイルがアップロードされる THEN システムはUTF-8、Shift-JIS、EUC-JPを自動検出する

### Requirement 9

**User Story:** アラーム管理者として、パース処理のパフォーマンスを確保したい。これにより、大きなCSVファイルでも快適に作業できる。

#### Acceptance Criteria

1. WHEN 10MB以上のCSVファイルがアップロードされる THEN システムは警告メッセージを表示する
2. WHEN 大きなファイルが処理される THEN システムはチャンク単位での処理を実行する
3. WHEN 処理に時間がかかる THEN システムは進行状況バーを表示する
4. WHEN メモリ使用量が上限に近づく THEN システムは処理を一時停止して警告を表示する
5. WHEN プレビュー表示が重い THEN システムは仮想スクロールを使用して表示パフォーマンスを最適化する

### Requirement 10

**User Story:** アラーム管理者として、エラー処理とユーザビリティを向上させたい。これにより、問題が発生した際に適切に対処できる。

#### Acceptance Criteria

1. WHEN CSVファイルの読み込みに失敗する THEN システムは具体的なエラー原因を表示する
2. WHEN ネットワークエラーが発生する THEN システムは再試行オプションを提供する
3. WHEN 設定変更中にページを離れようとする THEN システムは未保存の変更について警告する
4. WHEN 長時間の処理中にタイムアウトが発生する THEN システムは処理状況を保存して再開オプションを提供する
5. WHEN ユーザーが操作に迷う THEN システムはヘルプテキストとツールチップを適切な箇所に表示する

### Requirement 11

**User Story:** アラーム管理者として、CSVデータからアラーム情報を切り出すルールを設定したい。これにより、様々な形式のCSVファイルから正確にデータを抽出できる。

#### Acceptance Criteria

1. WHEN CSVデータが既知の形式（旧形式、新形式、拡張形式）にマッチする THEN システムは適切な切り出しルールを自動適用する
2. WHEN ユーザーがカスタム切り出しルールを作成する THEN システムは正規表現パターンと出力マッピングの設定インターフェースを提供する
3. WHEN 切り出しルールが設定される THEN システムはサンプルデータでルールをテストして結果をプレビュー表示する
4. WHEN 切り出しルールが適用される THEN システムはアラーム番号、アドレス種別、アドレス、ビット、コメントを正確に抽出する
5. WHEN 無効な切り出しルールが入力される THEN システムは具体的なエラーメッセージを表示して修正を促す
